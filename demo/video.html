The Problem

Counterparty onboarding exposes sensitive data
to every agent in the pipeline.

Traditional KYC/KYB sends raw PII through multiple systems. No proof of who accessed what. No cryptographic boundary. No way to revoke access after the decision.

How Verigate solves it

Agent acts on behalf of data owner — inside TEE

Counterparty delegates agent via W3C VC credential (EIP-191 signed). Agent executes inside T3N TEE — verifies credentials, assesses risk with AI, makes decision. PII never leaves the enclave.

Runtime Enforced

Without delegation grant: T3N returns 403 Forbidden. Agent cannot act without data owner's cryptographic authorization.

Enforcement

State Machine Rejects Violations

TEE contract enforces step ordering. If agent tries to skip verification and go directly to risk assessment — the state machine blocks execution and records the violation.

Not just logging — actual enforcement

T3N returns HTTP 400: "Out of order: expected 'verify-credential', got 'assess-risk'". Violation permanently recorded in evidence chain.

Privacy

Selective Disclosure — PII Never in Contract

Contract sends {{profile.first_name}} template placeholders. T3N host resolves them to actual PII outside the contract's memory. Contract never sees raw data.

Proven: PlaceholderUnknown error

T3N host attempted to resolve {{profile.company_name}} — confirms host processes templates. Raw PII never enters contract execution context.
